## Structure
```
# Requirements: <product name>

## Executive Summary
One paragraph: what the product does, for whom, and why it matters.

## Problem Statement
What problem exists today and what gaps the product fills.

## Goals
Bulleted list of measurable outcomes this product must achieve.

## Scope
What is explicitly in scope and explicitly out of scope for this release.

## Stakeholders
| Role | Name/Group | Interest |
|------|------------|----------|

## Assumptions
Bulleted list of assumptions made during requirements gathering.

## Constraints
Bulleted list of known technical, business, or regulatory constraints.

## User Personas
For each persona: name, role, goals, pain points.

## User Stories
- As a <persona>, I want <capability>, so that <benefit>

## Functional Requirements
| ID   | Requirement | Priority | Notes |
|------|-------------|----------|-------|
| FR-1 |             |          |       |

## Non-Functional Requirements

### Performance
| ID    | Requirement | Metric | Notes |
|-------|-------------|--------|-------|
| NFR-1 |             |        |       |

### Scalability
| ID    | Requirement | Metric | Notes |

### Security
| ID    | Requirement | Metric | Notes |

### Reliability
| ID    | Requirement | Metric | Notes |

### Availability
| ID    | Requirement | Metric | Notes |

### Maintainability
| ID    | Requirement | Metric | Notes |

### Accessibility
| ID    | Requirement | Metric | Notes |

### Usability
| ID    | Requirement | Metric | Notes |

### Logging & Monitoring
| ID    | Requirement | Metric | Notes |

### Error Handling
| ID    | Requirement | Metric | Notes |

### Backup & Recovery
| ID    | Requirement | Metric | Notes |

### Deployment Requirements
| ID    | Requirement | Metric | Notes |

### Browser/Platform Compatibility
| ID    | Requirement | Metric | Notes |

### Compliance
| ID    | Requirement | Metric | Notes |

### Localization
| ID    | Requirement | Metric | Notes |

## Acceptance Criteria
Grouped by feature area. Each criterion in Given/When/Then form.

## Risks
| ID | Risk | Probability | Impact | Mitigation |
|----|------|-------------|--------|------------|

## Success Metrics
Measurable KPIs that define success for this product.
```
